Fonctionnement des pull request 

~\newline


Une pull request est une sorte de commit attendant approbation. Afin que le repository soit le plus propre possible, vous allez donc devoir y avoir recours.

Comment faire une pull request ?

Il va tout d\textquotesingle{}abord devoir forker (copie du repos chez vous) le repository sur votre compte github, vous ferez les mo3difications sur celui-\/ci.



Pour demander l\textquotesingle{}ajout de votre travail sur le repository, vous allez faire une pull request depuis {\bfseries{votre fork}}.



Vous arrivez donc sur cette page qui repertorie tout vos changements, vous n\textquotesingle{}avez qu\textquotesingle{}a créer la pull request.



Pour finir décrivez vos changements.

(faites bien attention a ce que la case soit coché afin de pouvoir effectuer des modifications avant de potentiellement merge la pull request)



Voilà vous avez effectué votre pull request, il faut maintenant attendre qu\textquotesingle{}elle soit validée, ce qui sera fait lors de la prochaine session architecture à l\textquotesingle{}école.
